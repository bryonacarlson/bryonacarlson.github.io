Core Process Launcher

IT Ops Console Orchestrator

Developed to consolidate separate administrative tools and support scripts into a single, compiled Windows desktop application. This utility provides technicians with a clean interface that eliminates directory mapping errors, minimizes execution delays, and streamlines on-site troubleshooting calls.

Development & Verification Steps:

  • Consolidated Launch Scripting: Managed the integration of background processes within a native launcher wrapper, completely hiding command-line background flashes during execution to maintain a professional user experience.
  • WPF/XAML User Interface: Designed a modern, clean dark-mode layout using structural XML layout templates, ensuring clear visual separation of tools and responsive element scaling across local display resolutions.
  • Safe UI Threading Execution: Structured the script processing logic to execute resource-heavy tasks independently, ensuring that the main user dashboard interface never freezes or locks up while automated support routines run in the background.

Data Management Plane

IT Asset Manager (v13.4 Relational UI)

Built to replace an unreliable, single-user manual Excel spreadsheet tracker with an embedded, local relational database solution. The utility automatically queries local endpoint attributes and logs assets directly into a structured database.

Development & Verification Steps:

  • Relational SQLite Transition: Directed the migration of tracking variables into a standalone relational database engine via standard SQLite dynamic libraries, allowing rapid local querying without installing software engines.
  • Automated String Parsing: Configured string evaluation rules within the scanning module to parse system text fields, pulling computer names, active models, and vendor specs to organize asset inventories.
  • Schema Protection and Continuity: Designed the application data tables to isolate specific hardware profiles dynamically based on data contexts, preventing database schema validation failures or broken fields.

Infrastructure Protection Layer

Cryptographic Backup Pipeline

A state-aware backup automation tool designed to protect active scripting changes, deployment configurations, and local system documentation across external backup volumes. It optimizes transfer times and eliminates data duplication using strict validation rules.

Active File Handlers

Ensured stable execution limits by managing standard streaming memory boundaries. Configured the file sharing attributes safely to allow the automated engine to copy active system configuration logs even while files are open, preventing script crashes.

SHA-256 Verification

Built a cryptographic verification step that logs file hashes into a local ledger tracking file (`hashlist.json`). The backup engine validates SHA-256 signatures before and after running to confirm data integrity and only copy modified streams.

Automated System Isolation

Programmed a background data recovery loop that operates completely independently of human interaction. The process identifies critical local configuration files and moves them into secure, verified directories (`\SystemData\.assets`) to guarantee recovery.
